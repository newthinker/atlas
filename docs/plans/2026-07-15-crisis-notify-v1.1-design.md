# Cassandra 危机监控通知模板设计 v1.1（修订版）

> **基线**：`docs/plans/2026-07-14-crisis-notification-templates-design.md`（v1.0）+ sprint-020 已交付实现。
> **动机**：sprint-020 QA 跨视角对抗审查的 6 条设计反馈（W1/W2/W3/W4/I3/I4，见
> `.arcforge/archive/sprint-020-2026-07-15/docs/05-review/code-review-report.md`）。
> 共同主题：代码忠实于 v1.0，但文案在真实使用场景（凌晨推送、IM 预览截断、数据管道故障）下可能误导读者。
> **性质**：纯渲染层文案/条件修订。零接口、零签名、零 NotifyContext 字段变更，消息类型矩阵与家族框架不动。

## 0. 不变项

- 消息类型矩阵（7 类、触发条件、P0/P1/P2 分级）、两级家族、五段骨架。
- 页脚归属规则：`notifyFooter` 常量只挂结构化家族，速报家族不挂（§R5 的内联限定语不属页脚）。
- 禁词表（必然/一定/即将）、4096 上限、纯文本 SendText。
- `Messages(cfg, nc)` / `FormatIntradayAlert(...)` / `NotifyContext` / `Trend` 全部签名。

## 1. 新增设计原则（并入 v1.0 §7 约束）

1. **非预测从句显式化**：语义句凡含方向性结论（"抬升/恶化/缓解"），必须带显式"此为状态描述而非预测/不构成操作依据"从句；括注对冲（如"样本量小"）不满足本条。具体时间窗（"3–12 个月"类）从语义句中移除。
2. **✅ 仅限全清**：✅ 只用于异常区为空（七指标无 🔴🟡）的状态降级；异常区非空的降级用中性 🔽，首行动词用"状态回落"而非"状态解除"。
3. **判定输入变化必须溯源**：状态判定输入当日发生变化（指标新进入 STALE）且当日恰有状态变更时，状态变更消息必须携带溯源警示；解释性 P2 速报不得成为该信息的唯一载体。
4. **术语外化**：面向订阅者的文案不使用仅内部有意义的机制词（"共振计数"等），改用其对读者的含义（"计入触发判定"）。

## 2. 修订项

### R1（W1）STALE 被动降级溯源——双侧增强

**问题**：RED 触发指标转 STALE 会退出触发判定，可能当日被动降级；v1.0 降级消息骨架无差异信息，解释真相的 P2 速报优先级最低且后发。凌晨值守会把"✅ 状态解除"读成真实缓解。

**a) renderTransition 降级路径**：当 `nc.NewStale` 非空且其中存在断更前（`nc.PrevDay[ind].Status`）为 RED/AMBER 的指标时，尾注（"共持续 N 个评估日"行）之前插入警示行：

```
⚠ 注意：本次变更当日 hy_oas 数据断更（断更前为红），触发条件可能被动解除而非真实缓解，请人工核实。
```

- 多个符合条件的指标以顿号连接、按 AllIndicators 序：`hy_oas、sofr_effr 数据断更（断更前为红、黄）`（颜色列表与指标列表同序对应）。
- 仅列断更前为 RED/AMBER 者；断更前为绿的指标不触发警示（不构成被动解除）。
- 升级路径不加此行（升级不存在"被动解除"误读；断更信息仍由 P2 承载）。

**b) renderOpsAlert**：当该指标断更前为 RED/AMBER 时，第二行末尾追加：

```
⚠ 断更前为红且计入触发判定，今日若出现状态解除可能为被动解除，请人工核实。
```

数据来源均为既有 `nc.PrevDay`，零新管道。

### R2（W2）降级条件符号 + CRISIS→WATCH 语义句

**问题**：CRISIS 退出仅要求情绪层转绿，信用/流动性可能仍红；"✅ + 危机状态解除"在 IM 预览截断下读作"全清可进场"。

- **首行**：异常区为空 → `[P1] ✅ 状态解除 FROM → TO · MM-DD`（维持 v1.0）；异常区非空 → `[P1] 🔽 状态回落 FROM → TO · MM-DD`。判定复用 `splitZones` 的 abnormal 区。
- **语义句**（CRISIS→WATCH）改为：

```
情绪层连续 %d 个交易日回落至绿。危机状态退出，转入观察期；信用/流动性等其余层面可能仍异常，见下。
```

### R3（W3）WATCH→BREWING 语义句去预测感

**问题**："3–12 个月内系统性风险显著抬升"是具体时间窗 + 方向性结论的预测形句，对冲弱于 crisisSentence，且挂 P0。

改为：

```
信用与流动性双红共振。历史样本中此组合后系统性风险抬升比例显著（样本量小，可能失效）；此为状态描述而非预测，不构成操作依据。
```

（删去"3–12 个月"时间窗；补显式非预测从句，与 crisisSentence 对冲强度对齐。）

### R4（W4）非色彩状态间迁移文案

**问题**：`colorWord` 把 STALE/NO_DATA/季末抑制 都压成"白"，非色彩间迁移（如 STALE→季末抑制）渲染出自相矛盾的"转白（原白）"。

`diffLine` 状态迁移分支增加规则：**两侧均为非色彩状态**时，改用 `nonColorNote` 具体文案：

```
sofr_effr 转季末抑制（原数据断更(STALE)）
```

一侧色彩、一侧非色彩时维持 v1.0（"move 转白（原绿）"语义清晰，不改）。

### R5（I3）盘中速报去归因 + 内联限定语

**问题**：v1.0 §5.7 的"疑似 carry trade 快速平仓"是未经验证的因果叙事，且该消息是全系统唯一 P0+🚨+叙事+无页脚四特征叠加者，最像交易信号。

§5.7 示例改为：

```
[P0] 🚨 USD/JPY 盘中急跌 -3.4% · 07-18 14:30
现价 152.1（5 观测日前 157.5）· 系统状态 BREWING · 成因未核实，非交易信号。今日此告警不再重复。
```

- "成因未核实，非交易信号"是**内联事实限定语**，不是页脚：速报家族"不挂 notifyFooter"的规则不变。
- **测试连锁**：v1.0 测试用"非交易信号"子串区分家族归属，本修订后盘中速报也含该词——页脚归属断言必须改为按 `notifyFooter` 完整常量（HasSuffix）判定，不得再用子串。

### R6（I4）P2 速报术语外化

§5.6 示例中：

- "已标记 STALE 退出共振计数" → "已标记 STALE、不再计入触发判定"
- "恢复后自动回归" → "数据恢复后自动重新计入"

修订后完整示例（含 R1b 条件行的形态）：

```
[P2] 🔧 hy_oas 数据源断更 · 07-14
最后观测 07-09（滞后 5 日 > 阈值 4 日），已标记 STALE、不再计入触发判定；数据恢复后自动重新计入。持续超一周需检查 FRED 通道。
⚠ 断更前为红且计入触发判定，今日若出现状态解除可能为被动解除，请人工核实。
```

（断更前为绿时无第三行，即 v1.0 两行形态 + 措辞替换。）

## 4. 测试要点

1. R1a 警示行三条件独立否定：NewStale 空 / 断更前为绿 / 升级路径——均不出现警示行；多指标顿号连接用例。
2. R1b 条件行：断更前红/黄出现、断更前绿不出现（恰好落界：AMBER 出现）。
3. R2 条件符号恰好落界：异常区恰为空 → ✅ 状态解除；恰有一个 🟡 → 🔽 状态回落。
4. R3/R2 语义句逐字对本设计（含"此为状态描述而非预测"从句）。
5. R4：STALE→季末抑制、NO_DATA→STALE 两个双非色彩用例；色彩↔非色彩维持旧断言。
6. R5：盘中速报含"成因未核实，非交易信号"、不含"carry trade"；**全家族页脚归属断言改为 HasSuffix(notifyFooter)**，确认速报两类不以 notifyFooter 结尾。
7. 禁词全家族回归（措辞变更后重跑）；4096 上限（R1a 警示行加长后的最坏组合）。

## 5. 向后兼容说明

纯文案/渲染条件变更，无数据结构、存储、配置项变化；订阅者无需任何操作。历史消息不回溯修改。
